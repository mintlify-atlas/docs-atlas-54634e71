Every warehouse in Bodega de Frío is divided into three zones. Each zone holds a different stage of a box's lifecycle.

## Data types

### Slot

A `Slot` represents one of the 12 positions in the **Bodega** zone.

```typescript
type Slot = {
  position: number;       // 1 – 12
  autoId: string;         // BOX-YYYYMMDD-NNN, empty if unoccupied
  name: string;           // box label
  temperature: number | null;
  client: string;         // client identifier
};
```

### Box

A `Box` is used in the **Ingresos** and **Salida** zones.

```typescript
type Box = {
  position: number;       // auto-incremented within the zone
  autoId: string;         // BOX-YYYYMMDD-NNN
  name: string;
  temperature: number;
  client: string;
};
```

### autoId format

Every box receives an `autoId` generated by `createAutoId`. The format is:

```
{PREFIX}-YYYYMMDD-NNN
```

where `NNN` is a zero-padded daily counter that resets each calendar day. The prefix depends on the registration type:

- `CAJ-YYYYMMDD-NNN` — for boxes registered via the inbound (Ingresos) form, using `createAutoId("CAJ")`
- `BOX-YYYYMMDD-NNN` — for boxes normalized from Firestore when no prefix is present

For example: `CAJ-20260326-001`.

## Registering a box (inbound)

Only the **custodio** role can register new boxes into the **Ingresos** zone.

<Steps>
  <Step title="Open the Ingresos tab">
    Log in as **custodio** and navigate to the **Ingresos** tab.
  </Step>
  <Step title="Fill in the form">
    Enter the box **name**, its **temperature** in °C, and select the **client** from the dropdown. The position in the Ingresos queue is assigned automatically — it is always the next available integer.
  </Step>
  <Step title="Submit">
    Click the register button. The action:
    - Generates a new `autoId` using `createAutoId("CAJ")` — e.g. `CAJ-20260326-001`.
    - Appends the box to `inboundBoxes`, sorted by position.
    - Increments `stats.ingresos`.
    - Records the box in the history document via `addIngreso`.
  </Step>
</Steps>

<Warning>
  The name field must not be empty, and the temperature must be a valid number. Both are validated before submission.
</Warning>

## Moving a box between positions

Box movements between zones are always driven by orders created by the **jefe** or **administrador** and executed by the **operario**. See [Order flow](/features/order-flow) for details.

Direct slot-to-slot moves within **Bodega** are also triggered through `a_bodega` orders. When the operario executes the order:

1. The source slot is cleared (`autoId`, `name`, `temperature`, `client` reset to empty).
2. The target slot receives the box data.
3. `stats.movimientosBodega` is incremented.

## Removing / dispatching a box

Only the **custodio** can dispatch a box from the **Salida** zone.

<Steps>
  <Step title="Locate the box in Salida">
    The **Salida** zone lists all outbound boxes. Identify the one to dispatch.
  </Step>
  <Step title="Click dispatch">
    The `handleDispatchBox` action removes the box from `outboundBoxes` and appends it to `dispatchedBoxes`. No order is required for this step.
  </Step>
</Steps>

<Note>
  Dispatched boxes are preserved in `dispatchedBoxes` for reporting purposes. They appear in the **Despachados** section and are counted in daily stats.
</Note>

## Searching by box ID

The **administrador** role has access to the global search bar in the dashboard header.

- Enter an `autoId` (e.g. `BOX-20260326-001`) or a box **name**.
- The `handleSearch` action scans `inboundBoxes`, `slots`, and `outboundBoxes` in that order.
- The dashboard displays the zone and position where the box was found, or a "not found" message.

```typescript
// Search checks all three zones in order
const ingreso = inboundBoxes.find(box => box.autoId === id || box.name === id);
const bodega  = slots.find(slot => slot.autoId === id || slot.name === id);
const salida  = outboundBoxes.find(box => box.autoId === id || box.name === id);
```

## Slot grid

The **Bodega** zone always contains exactly 12 slots (positions 1 – 12). Unoccupied slots have an empty `autoId`. The occupied count is displayed in the dashboard header.

<Tip>
  The slot grid is rendered by `EstadoBodegaSection`. Clicking a slot opens a detail view showing the box name, temperature, client, and `autoId`.
</Tip>
