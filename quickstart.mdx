---
title: "Quick start"
description: "Set up Bodega de Frío locally and log in for the first time."
---

This guide takes you from a fresh clone to a running local instance connected to your Firebase project.

<Note>
  Bodega de Frío requires a Firebase project with Firestore and Email/Password Authentication enabled. All users must be created by an administrator or configurador — there is no self-registration.
</Note>

<Steps>
  <Step title="Clone the repository and install dependencies">
    ```bash
    git clone https://github.com/eldani13/frio.git
    cd frio
    npm install
    ```
  </Step>

  <Step title="Create a Firebase project">
    1. Go to [console.firebase.google.com](https://console.firebase.google.com) and create a new project.
    2. In **Build → Firestore Database**, create a database (start in test mode for development).
    3. In **Build → Authentication**, enable the **Email/Password** sign-in provider.
    4. In **Project settings → Your apps**, register a **Web app** and copy the Firebase config values.

    See [Firebase setup](/configuration/firebase-setup) for detailed instructions.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env.local` file at the project root:

    ```bash .env.local
    NEXT_PUBLIC_FIREBASE_API_KEY=your-api-key
    NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
    NEXT_PUBLIC_FIREBASE_PROJECT_ID=your-project-id
    NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your-project.appspot.com
    NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your-sender-id
    NEXT_PUBLIC_FIREBASE_APP_ID=your-app-id
    NEXT_PUBLIC_WAREHOUSE_ID=default
    ```

    <Warning>Never commit `.env.local` to version control.</Warning>

    See [Environment variables](/configuration/environment-variables) for the full variable reference including optional Fridem integration variables.
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser.
  </Step>

  <Step title="Log in and explore the dashboard">
    You will see the login screen. Enter the email and password of a user that has been created in Firebase Authentication and has a corresponding profile document in Firestore (`users/{uid}` or `usuarios/{uid}` with a `role` field).

    After logging in:
    1. The **WarehouseSelector** appears — choose a warehouse from the list (a `default` warehouse is created automatically if none exists).
    2. Use the dashboard tabs to navigate: **Estado**, **Ingresos**, **Salida**, **Órdenes**, **Despachados**, **Reportes**, and **Configuración** (available tabs depend on your role).

    <Tip>
      For a first test, log in with a **configurador** role account. This gives you access to the **Configuración** tab where you can create warehouses, clients, and additional users.
    </Tip>
  </Step>
</Steps>

## Available scripts

| Script | Description |
|--------|-------------|
| `npm run dev` | Start Next.js in development mode with hot reload |
| `npm run build` | Build the application for production |
| `npm run start` | Start the production build |
| `npm run lint` | Run ESLint |
