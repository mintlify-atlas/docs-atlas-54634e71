---
title: "Bodega cloud state"
description: "Functions for reading and writing warehouse state to Firestore."
---

The `bodegaCloudState` module provides functions for syncing warehouse state to and from Firestore in real time. All functions target two fixed documents per warehouse: `state/main` for live operational state and `state/history` for historical records.

## Firestore paths

| Document | Path |
|---|---|
| Main state | `warehouses/{warehouseId}/state/main` |
| History state | `warehouses/{warehouseId}/state/history` |

## Constants

### DEFAULT_WAREHOUSE_ID

```typescript
export const DEFAULT_WAREHOUSE_ID = process.env.NEXT_PUBLIC_WAREHOUSE_ID ?? "default";
```

The fallback warehouse ID used when `NEXT_PUBLIC_WAREHOUSE_ID` is not set in the environment. Pass this constant to any of the functions below when you do not need to target a specific warehouse dynamically.

## CloudWarehouseState type

The shape of a warehouse's live state document in Firestore:

```typescript
export type CloudWarehouseState = {
  slots: Slot[];
  inboundBoxes: Box[];
  outboundBoxes: Box[];
  dispatchedBoxes: Box[];
  orders: BodegaOrder[];
  stats: BodegaStats;
  warehouseName: string;
  alerts: AlertItem[];
  assignedAlerts: AlertAssignment[];
  alertasOperario: Array<{ position: number; [key: string]: unknown }>;
  alertasOperarioSolved: number[];
  llamadasJefe: Array<Record<string, unknown>>;
};
```

<ResponseField name="slots" type="Slot[]">
  Current storage slot occupancy records.
</ResponseField>

<ResponseField name="inboundBoxes" type="Box[]">
  Boxes in the inbound (receiving) zone.
</ResponseField>

<ResponseField name="outboundBoxes" type="Box[]">
  Boxes staged for dispatch.
</ResponseField>

<ResponseField name="dispatchedBoxes" type="Box[]">
  Boxes that have been dispatched out of the warehouse.
</ResponseField>

<ResponseField name="orders" type="BodegaOrder[]">
  Active warehouse movement orders.
</ResponseField>

<ResponseField name="stats" type="BodegaStats">
  Aggregate counters for ingresos, salidas, and movimientosBodega.
</ResponseField>

<ResponseField name="warehouseName" type="string">
  Display name of the warehouse.
</ResponseField>

<ResponseField name="alerts" type="AlertItem[]">
  Unresolved alerts generated by warehouse operations.
</ResponseField>

<ResponseField name="assignedAlerts" type="AlertAssignment[]">
  Alerts that have been assigned to a staff member for resolution.
</ResponseField>

<ResponseField name="alertasOperario" type="Array">
  Operator-facing alert entries, each carrying at minimum a `position` number.
</ResponseField>

<ResponseField name="alertasOperarioSolved" type="number[]">
  Positions of operator alerts that have been marked resolved.
</ResponseField>

<ResponseField name="llamadasJefe" type="Array">
  Call-to-supervisor events logged during operations.
</ResponseField>

## Main state functions

### ensureWarehouseState

```typescript
export async function ensureWarehouseState(warehouseId: string): Promise<void>
```

Checks whether the `state/main` document exists for the given warehouse and creates it with `defaultWarehouseState` if it does not. Uses `serverTimestamp()` for the `createdAt` field on first creation.

Call this function once during application initialization so that subsequent reads and subscriptions always find an existing document.

<ParamField path="warehouseId" type="string" required>
  The Firestore document ID of the warehouse, e.g. `"GENERAL"` or the value of `DEFAULT_WAREHOUSE_ID`.
</ParamField>

---

### subscribeWarehouseState

```typescript
export function subscribeWarehouseState(
  warehouseId: string,
  handler: (state: CloudWarehouseState) => void,
): () => void
```

Attaches a real-time Firestore `onSnapshot` listener to `warehouses/{warehouseId}/state/main`. Every time the document changes, `handler` is called with the merged state (falling back to `defaultWarehouseState` for any missing fields).

Internally calls `ensureWarehouseState` before setting up the listener, so you do not need to call it separately.

<ParamField path="warehouseId" type="string" required>
  The Firestore document ID of the warehouse.
</ParamField>

<ParamField path="handler" type="(state: CloudWarehouseState) => void" required>
  Callback invoked on every snapshot update. Receives the full `CloudWarehouseState`, with missing Firestore fields replaced by their defaults.
</ParamField>

<ResponseField name="returns" type="() => void">
  An unsubscribe function. Call it when the component or module that established the subscription unmounts or is destroyed to stop receiving updates and avoid memory leaks.
</ResponseField>

```typescript
import { subscribeWarehouseState, DEFAULT_WAREHOUSE_ID } from "@/lib/bodegaCloudState";

const unsubscribe = subscribeWarehouseState(DEFAULT_WAREHOUSE_ID, (state) => {
  console.log(state.slots);
});

// Later, when cleaning up:
unsubscribe();
```

---

### saveWarehouseState

```typescript
export async function saveWarehouseState(
  warehouseId: string,
  state: Partial<CloudWarehouseState>,
): Promise<void>
```

Persists a partial `CloudWarehouseState` to the `state/main` document using a merge `setDoc` call. Only the fields you provide are written; all other fields in the document remain untouched. Firestore's `serverTimestamp()` is automatically written to `updatedAt` on every save.

<ParamField path="warehouseId" type="string" required>
  The Firestore document ID of the warehouse.
</ParamField>

<ParamField path="state" type="Partial<CloudWarehouseState>" required>
  An object containing only the fields you want to update. Unspecified fields are left unchanged in Firestore.
</ParamField>

```typescript
import { saveWarehouseState, DEFAULT_WAREHOUSE_ID } from "@/lib/bodegaCloudState";

await saveWarehouseState(DEFAULT_WAREHOUSE_ID, {
  warehouseName: "Bodega Norte",
  stats: { ingresos: 12, salidas: 4, movimientosBodega: 8 },
});
```

<Note>
  `saveWarehouseState` does not verify that the document exists before writing. If you are unsure whether the document has been initialized, call `ensureWarehouseState` first or use `subscribeWarehouseState`, which handles initialization automatically.
</Note>

## History state functions

The history document (`state/history`) stores arrays of completed ingress, egress, movement, and alert records. Its shape is defined by `HistoryState`:

```typescript
export type HistoryState = {
  ingresos: Box[];
  salidas: BodegaOrder[];
  movimientosBodega: BodegaOrder[];
  alertas: AlertHistoryEntry[];
};
```

---

### ensureHistoryState

```typescript
export async function ensureHistoryState(warehouseId: string): Promise<void>
```

Checks whether the `state/history` document exists and creates it with `defaultHistoryState` if it does not. Mirrors the behavior of `ensureWarehouseState` for the history document.

<ParamField path="warehouseId" type="string" required>
  The Firestore document ID of the warehouse.
</ParamField>

---

### subscribeHistoryState

```typescript
export function subscribeHistoryState(
  warehouseId: string,
  handler: (state: HistoryState) => void,
): () => void
```

Attaches a real-time Firestore listener to `warehouses/{warehouseId}/state/history`. Calls `ensureHistoryState` internally before subscribing.

<ParamField path="warehouseId" type="string" required>
  The Firestore document ID of the warehouse.
</ParamField>

<ParamField path="handler" type="(state: HistoryState) => void" required>
  Callback invoked on every snapshot update. Receives the full `HistoryState`, with missing fields replaced by their defaults (empty arrays).
</ParamField>

<ResponseField name="returns" type="() => void">
  An unsubscribe function. Call it to detach the listener.
</ResponseField>

---

### saveHistoryState

```typescript
export async function saveHistoryState(
  warehouseId: string,
  state: Partial<HistoryState>,
): Promise<void>
```

Persists a partial `HistoryState` to the `state/history` document using a merge `setDoc` call. Writes `serverTimestamp()` to `updatedAt` on every call.

<ParamField path="warehouseId" type="string" required>
  The Firestore document ID of the warehouse.
</ParamField>

<ParamField path="state" type="Partial<HistoryState>" required>
  An object containing only the history arrays you want to update.
</ParamField>

```typescript
import { saveHistoryState, DEFAULT_WAREHOUSE_ID } from "@/lib/bodegaCloudState";

await saveHistoryState(DEFAULT_WAREHOUSE_ID, {
  ingresos: updatedIngresosList,
});
```
